# Technical Journaling Format

Each week of the bootcamp gets one journal entry in this folder, using the
sections below.

## Technical Goal
The technical goal planned for this week of the bootcamp.

## Technical Uncertainty
What is uncertain in terms of technical achievement, and could result in
technical failure or a dead end.

## Technical Hypotheses
An educated guess on what the technical outcome will be.

## Technical Observations
Key observations from the implementation work, useful to someone trying to
replicate the experience. Large documentation is summarized and linked; proof
of work (screenshots, code samples, benchmarks) belongs here. Routine work is
omitted.

## Technical Conclusions
Reflecting back on the hypotheses: what were the outcomes? Any new uncertainty
set aside for future exploration, next steps, or considerations worth noting.

## Key Takeaway
One sentence: the most important lesson of the week.
